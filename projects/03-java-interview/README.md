## 一、系统设计（必考，3 题）

### 1.1 高并发秒杀

```
CDN 静态化 → 网关限流 → 队列削峰 → Redis 预扣库存 → 异步下单 → DB 扣减
```

**2 层追问准备**：
- 超卖怎么防？→ Lua 脚本原子扣减 + DB 对账兜底
- 热点商品怎么处理？→ 本地缓存预热 + 队列隔离

**口述 5 分钟**：从请求进来到 DB 落库，每层干什么、为什么这样设计。

### 1.2 短链系统

```
短链生成 → 存储映射 → 访问重定向 → 统计分析
```

**核心考点**：发号器方案（雪花 ID vs Hash 冲突）、热点短链缓存策略。

### 1.3 设计一个配置中心

```
配置变更 → 长轮询通知 → 客户端热更新 → 版本回滚
```

**考察点**：灰度推送策略、配置版本管理。

## 二、核心组件深度（高频追问）

### 2.1 Redis

| 场景 | 常见追问 | 技术要点 |
|------|---------|------------|
| 缓存穿透 | 布隆过滤器和空值缓存的区别？ | 布隆过滤器有误判、空值缓存占内存，大数据量用布隆 |
| 缓存击穿 | 互斥锁和逻辑过期的取舍？ | 互斥锁简单但阻塞，逻辑过期复杂但性能好 |
| 雪崩 | 多级缓存怎么设计？ | 本地 Caffeine + Redis + DB，TTL 随机化 |
| 分布式锁 | Redisson 看门狗原理？ | 锁续期机制，默认 30s，每 10s 续一次 |
| 限流 | 令牌桶 vs 滑动窗口怎么选？ | 令牌桶允许突发，滑动窗口精度高但内存大 |

### 2.2 MySQL

| 考点 | 准备 |
|------|------|
| 索引优化 | 联合索引最左前缀 + 覆盖索引 + EXPLAIN 解读 |
| SQL 优化 | 慢查询定位 → type/rows/Extra → 加索引 or 改写法 |
| 分库分表 | ShardingSphere 分片键设计 + 跨分片查询限制 |
| 实战案例 | 准备一个你调过的慢查询（现象 → 根因 → 优化 → 效果数据） |

### 2.3 RocketMQ（供应链/物流方向 JD 高频，重点）

| 考点 | 要点 |
|------|------|
| 可靠性 | 同步刷盘 + 生产者确认 + 消费者手动 ack |
| 顺序消息 | 同 MessageGroup 发到同一队列，消费者串行处理 |
| 幂等 | 业务唯一键 + DB 唯一索引 |
| 事务消息 | 半消息 → 执行本地事务 → 提交/回滚 + 回查机制 |
| 堆积 | 扩容 Consumer → 降级非核心 → 积压告警 |

### 2.4 JVM

- 内存模型：堆/栈/元空间/直接内存
- GC：G1 原理（Region + 混合回收）、什么时候用 ZGC
- 调优：准备一个案例 — 现象(Full GC 频繁) → jstat/jmap 定位 → 调参 → 效果
- 常用参数：-Xmx/-Xms、GC 日志、heap dump

## 六、业务场景靶向

### 车联网方向

- 业务特征：智驾/座舱后端 + 车联网数据平台
- 技术栈：Spring Cloud + MQTT(车端通信) + Kafka/Flink(数据处理) + 时序DB
- 考点偏向：算法手写 + Spring Cloud 全栈 + Redis + JVM
- 设计题：「车联网数据采集平台」— MQTT 网关 → 消息队列 → 流计算 → 时序DB

### 电商供应链方向

- 业务特征：WMS/TMS/前置仓/配送
- 技术栈：Spring Cloud + RocketMQ + MySQL(ShardingSphere) + Redis
- 考点偏向：RocketMQ 深度 + MySQL 优化 + 分布式事务 + 供应链架构
- 设计题：「生鲜库存系统」— 库存模型 + 实时同步 + 效期管理

### 物流平台方向

- 业务特征：零担物流 + 路由调度 + 运力匹配
- 技术栈：Spring Cloud + Kafka + Redis + MySQL + 路由算法
- 考点偏向：Spring Cloud + Kafka + 高并发 + 平台架构设计
- 设计题：「物流路由调度系统」— 路径规划 + 中转 + 实时轨迹

### 海外电商方向

- 业务特征：手机电商 + 多语言/多币种 + 跨境部署
- 技术栈：Spring Boot + MySQL + Redis + MQ + 海外云
- 考点偏向：分布式基础 + 多区域部署 + 数据同步
- 设计题：「多区域电商部署策略」— 区域部署 + 数据同步 + CDN
